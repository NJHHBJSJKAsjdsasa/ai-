## 模块说明

### 1. 用户接口层 (Interface)

#### CLI (`interface/cli.py`)
- **职责**：提供命令行交互界面
- **功能**：
  - 用户输入处理
  - 命令解析（/help, /quit, /memories 等）
  - 彩色输出显示
  - 输入历史记录
  - 加载动画

#### Web API (`interface/web_api.py`)
- **职责**：提供 HTTP RESTful API
- **功能**：
  - FastAPI 应用实例
  - 记忆 CRUD 接口
  - 统计信息接口
  - 健康检查接口
  - CORS 跨域支持

### 2. 核心业务层 (Core)

#### Memory Manager (`core/memory.py`)
- **职责**：管理记忆的完整生命周期
- **核心方法**：
  - `add_memory()`: 添加新记忆（包含隐私检测、重要性评分）
  - `get_memory()`: 获取记忆（自动更新访问次数）
  - `get_all_memories()`: 获取所有记忆
  - `search_memories()`: 搜索记忆
  - `update_memory()`: 更新记忆
  - `delete_memory()`: 删除记忆
  - `_calculate_retention_days()`: 根据重要性计算保留天数

#### Selector (`core/selector.py`)
- **职责**：评估内容重要性，决定是否值得记忆
- **评分维度**：
  - 内容长度评分（5-500字符为佳）
  - 问题检测评分（疑问词、问号）
  - 个人信息检测（第一人称词汇）
  - 数字/日期检测
  - 情感强度评分
- **分类功能**：
  - question（问题）
  - personal_info（个人信息）
  - fact（事实）
  - chat（闲聊）

#### Forgetter (`core/forgetter.py`)
- **职责**：管理记忆的遗忘和归档
- **核心功能**：
  - `is_expired()`: 检查记忆是否过期
  - `calculate_memory_strength()`: 计算记忆强度
  - `should_forget()`: 判断是否该遗忘
  - `forget_expired_memories()`: 遗忘过期记忆
  - `archive_old_memories()`: 归档旧记忆
  - `strengthen_memory()`: 强化记忆（延长保留期）
- **记忆强度公式**：
  ```
  strength = importance_factor * 0.4 + access_factor * 0.3 + recency_factor * 0.3
  ```

#### Judge (`core/judge.py`)
- **职责**：综合评估内容，做出学习、存储、加密决策
- **评估维度**：
  - **隐私风险评估** (0-100)：检测银行卡、身份证号、手机号、邮箱等
  - **安全风险评估** (0-100)：检测攻击性内容、敏感话题、恶意代码、SQL注入、XSS等
  - **内容质量评估** (0-100)：评估长度、语法、信息量
  - **价值评估** (0-100)：评估实用价值、情感价值、未来价值
- **决策输出**：
  - `should_learn`: 是否适合学习
  - `should_store`: 是否值得存储
  - `should_encrypt`: 是否需要加密
  - `risk_level`: 风险等级（critical/high/medium/low/minimal）

### 3. 数据存储层 (Storage)

#### Database (`storage/database.py`)
- **职责**：SQLite 数据库操作
- **核心功能**：
  - 数据库连接管理
  - 表结构创建
  - CRUD 操作
  - 搜索查询
  - 统计功能
- **表结构**：
  ```sql
  CREATE TABLE memories (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      content TEXT NOT NULL,
      importance INTEGER NOT NULL DEFAULT 5,
      category TEXT NOT NULL DEFAULT 'general',
      created_at TEXT NOT NULL,
      last_accessed TEXT NOT NULL,
      access_count INTEGER NOT NULL DEFAULT 0,
      privacy_level INTEGER NOT NULL DEFAULT 50,
      is_encrypted INTEGER NOT NULL DEFAULT 0,
      retention_days INTEGER NOT NULL DEFAULT 30
  );
  ```

#### Models (`storage/models.py`)
- **职责**：数据模型定义
- **Memory 类属性**：
  - `id`: 记忆ID
  - `content`: 内容
  - `importance`: 重要性 (0-10)
  - `category`: 分类
  - `created_at`: 创建时间
  - `last_accessed`: 最后访问时间
  - `access_count`: 访问次数
  - `privacy_level`: 隐私级别 (0-100)
  - `is_encrypted`: 是否加密
  - `retention_days`: 保留天数

### 4. 工具层 (Utils)

#### Colors (`utils/colors.py`)
- **职责**：终端颜色输出
- **功能**：
  - 颜色常量定义
  - 文本着色函数
  - 加载动画 (Spinner)

#### Logger (`utils/logger.py`)
- **职责**：日志记录
- **功能**：
  - 多级别日志（DEBUG, INFO, WARNING, ERROR）
  - 控制台输出
  - 文件输出

#### Privacy Detector (`utils/privacy_detector.py`)
- **职责**：敏感信息检测和加密
- **检测类型**：
  - 手机号
  - 邮箱
  - 身份证号
  - 银行卡号
  - 密码
- **功能**：
  - 敏感信息检测
  - 自动加密

#### Scheduler (`utils/scheduler.py`)
- **职责**：定时任务调度
- **功能**：
  - 定时遗忘任务
  - 定时归档任务

## 设计原则

1. **单一职责原则**：每个模块只负责一个明确的功能
2. **依赖注入**：通过构造函数注入依赖，便于测试和替换
3. **开闭原则**：对扩展开放，对修改关闭
4. **数据驱动**：所有决策基于数据评估，避免硬编码规则
5. **安全第一**：隐私和安全评估优先于其他评估
